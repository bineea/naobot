# naobot 产品能力审计（2026-07-14）

## 审计目的

本报告记录当前 HEAD 的产品能力快照。naobot 的目标不是停留在软件 MVP，而是成为面向家庭多人、默认儿童安全、高度主动、功能完善且长期有趣的桌面陪伴机器人。

本报告负责记录“现在做到什么程度”；长期目标见 `docs/product/prd.md`，推进顺序见 `docs/product/roadmap.md`，完成定义见 `docs/product/acceptance.md`。

## 成熟度定义

| 等级 | 定义 |
| --- | --- |
| L0 缺失 | 没有可运行实现，或只有产品设想。 |
| L1 原型 | 存在接口、骨架、demo 或局部实现，尚未形成可信闭环。 |
| L2 自动化验证 | 关键逻辑有自动化测试或 fake 验证，但未完成真实软硬件闭环。 |
| L3 集成可用 | 在目标软件环境和真实设备链路上形成端到端闭环。 |
| L4 场景验收 | 通过家庭多人、儿童安全和连续使用场景验收。 |
| L5 产品稳定 | 具备部署、升级、恢复、监控和长期稳定性证据。 |

任何能力只有在达到对应等级的全部证据后才能升级，不能用 mock、协议测试或代码存在代替真实硬件和家庭场景验收。

## 能力成熟度总览

| 产品能力域 | 当前等级 | 已有基础 | 主要缺口 |
| --- | --- | --- | --- |
| 陪伴与关系 | L2 | 人物注册、AgentScope runtime 隔离、Soul、待确认 Memory | Memory 仍是全局 JSON，缺少个人/家庭共享边界、关系成长和监护人模型 |
| 自然交流 | L2 | 控制/媒体双链路、VAD、ASR/TTS adapter、会话触发和半双工 | 真实 provider 未形成开箱闭环，对视判断不真实，音频格式与采样率契约需加固 |
| 生命感与主动性 | L1 | 触摸、问候、目光等触发入口和语义行为编译 | 缺少动机、精力、兴趣、打扰预算、冷却、主动行为仲裁和反馈学习 |
| 身体表达 | L2 | 圆眼表情、参数化 expression、动作白名单、可中断 MotionController | 表情参数未完整生效，显示/蜂鸣器仍有阻塞，步态缺少校准和实机调优 |
| 趣味系统 | L1 | 基础动作、表情、对话和 Routine CRUD | 缺少可扩展技能包、玩法状态机、生成式编排、内容复用和长期新鲜度机制 |
| 受限桌面探索 | L0 | 仅有小步和转向技能 | 没有桌边/距离感知、区域约束、定位、避障、探索仲裁和安全验收，必须保持禁用 |
| 家庭与儿童安全 | L1 | 身份隔离、显式注册、数据删除、动作安全边界 | 缺少监护人权限、儿童内容分级、个人/共享记忆策略、陌生人模式和隐私退出机制 |
| 可靠性与产品化 | L1 | 自动化测试、心跳、失联降级、诊断页面和固件构建配方 | 触摸/电量仍为 stub，无定制实测 bin、OTA、配网、看门狗、校准和长期硬件记录 |
| AI 与扩展生态 | L2 | AgentScope 自动路由、多 Agent、持久 runtime、确定性 L2 编译 | 缺少正式技能注册/版本协议、能力发现、评测集和安全的第三方扩展机制 |

## P0：产品成立前必须解决

- 实现真实触摸输入、去抖和事件语义；当前 `firmware/esp32/hardware/touch.py` 仍是状态 stub。
- 实现电池 ADC、电压曲线、充电状态和低电阈值；当前固定电量无法触发真实低电反射。
- 打通控制链路的 `touch_head` 与媒体注册确认，避免 Host 等待媒体控制消息而固件只在控制 WebSocket 上报事件。
- 将 Memory 从全局数据改为按人物、家庭共享域和儿童策略隔离；禁止跨人物自动注入全部长期记忆。
- 建立监护人、儿童、成人、访客权限和数据删除模型；儿童默认最小化持久化。
- 在真实 N16R8 上完成定制固件、摄像头、音频、OLED、MPU6050、触摸、电量和舵机闭环验收。
- 受限桌面探索增加向下边缘检测和前向距离感知前保持关闭，不能依靠摄像头或 MPU6050 猜测桌边。

## P1：完整陪伴体验的关键缺口

- 用真实眼部方向和连续时长替代“检测到单张人脸即对视 1.5 秒”的占位逻辑。
- 统一 ASR/TTS PCM16 16 kHz 契约：上传标准容器或明确 raw 参数，下行必须校验、重采样并限幅。
- 让 `conversation_sessions` 进入生产会话生命周期，保存触发源、人物、起止时间和终止原因。
- 增加 Routine 触发与执行引擎，而不只是创建、确认和删除。
- 增加主动行为运行时：动机、情绪、精力、兴趣、打扰预算、勿扰、冷却和用户反馈。
- 将 LLM 创作限制为对话、expression 和安全技能编排，由 L2 编译器做时长、资源、重复度和儿童内容校验。
- 修复 Host fault/low-battery 状态恢复、固件 reflex recovered 生命周期和恢复事件。
- 将 OLED 动画、蜂鸣器和剩余阻塞动作改为可 tick/cancel，保证安全循环不被数百毫秒 sleep 阻塞。
- 将 intent 回执拆分为 accepted/completed/failed，增加去重、重放保护和有界动作队列。
- 提供联网优先的一键配置路径、`.env` 示例、provider 健康检查和明确的降级体验。

## P2：质量与可维护性增强

- 完整使用 `valence/arousal/eye_open/pupil_offset_x/blink_rate/duration_ms`，形成连续而非枚举映射的眼睛表现。
- 增强 IMU 滤波、自由落体检测、姿态恢复状态机和异常校准保护。
- 增加每舵机方向、零位、机械限位、速度和步态参数校准。
- 降低单 Agent 升级团队的最坏延迟，并允许单个专家失败时由负责人使用其余结果收敛。
- 完善 Dashboard 的全动作测试、Soul 字段、Routine 创建、诊断导出和家庭管理页面。
- 增强模拟器/CLI 的 heartbeat 跳过、token 传递和完整 ACK 生命周期。
- 增加设备/管理员分权、速率限制、局域网 TLS 方案和密钥轮换。
- 增加 CI、覆盖率、类型检查、依赖锁定、固件 worker supervisor 和硬件验收日志采集器。
- 将 WiFi、设备 token 和板级配置从冻结源码迁移到安全配置流程。

## 正式产品拓展方向

以下能力不再视为可有可无的远期想法，而是完整产品路线的一部分：

- 家庭关系图谱、个人记忆与家庭共享记忆、关系成长和可解释遗忘。
- 高度主动的陪伴行为，但受勿扰、频率、儿童安全和场景置信度约束。
- 安全技能包与 LLM 即兴创作结合的故事、模仿、小游戏、庆祝和日常仪式。
- 可组合的表情、声音、姿态和动作编排，以及重复度控制和个性化偏好学习。
- 在专用边缘/距离传感器、区域约束和实机安全验收基础上的受限桌面探索。
- 配网、OTA、配置备份、故障恢复、设备健康、自动校准和长期运行监控。

## 推荐推进顺序

```text
真实感知与可信身体
-> 家庭身份、记忆与儿童安全
-> 自然连续交流闭环
-> 主动陪伴与人格成长
-> 生成式趣味技能系统
-> 安全的受限桌面探索
-> 部署、升级与长期稳定性
```

## 当前证据边界

- 当前自动化测试证明的是软件逻辑和 fake 环境，不证明 L3 以上产品成熟度。
- generic MicroPython bin 不包含项目定制 camera 模块，不能作为媒体固件验收产物。
- 没有带时间戳的真实 N16R8 连续运行、家庭试用和儿童安全记录，因此当前没有能力达到 L4 或 L5。
- 本报告应在关键阶段验收后新增下一份日期化审计，不覆盖历史报告。
