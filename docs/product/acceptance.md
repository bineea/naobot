# 验收标准

## 完成定义

naobot 不以“软件模块齐全”作为产品完成标准。每项能力按以下成熟度独立验收：

| 等级 | 必要证据 |
| --- | --- |
| L1 原型 | 可运行接口、demo 或局部行为，已明确限制和 fallback。 |
| L2 自动化验证 | 单元、协议或 fake 集成测试通过，非法输入和失败路径有覆盖。 |
| L3 集成可用 | 真实 Host、N16R8、传感器和执行器完成端到端闭环，并保留日志。 |
| L4 家庭场景验收 | 成人、儿童、访客、多人物切换和连续使用场景通过。 |
| L5 产品稳定 | 配网、升级、回滚、恢复、监控和长期稳定性通过发布门槛。 |

低等级证据不能替代高等级验收。当前状态见 `docs/reviews/2026-07-14-product-capability-audit.md`。

## 自动化验证门槛（L2）

- AgentScope 覆盖流式输出、非法输出/异常/超时 fallback、自动路由、低置信度升级、多 Agent 收敛和安全事件不组队。
- Runtime 覆盖 SQLite WAL、人物隔离、访客销毁、并发锁、reset/delete、媒体清洗和进程重启恢复。
- 控制和媒体协议覆盖鉴权、限长、分片、坏帧、背压、heartbeat、intent 安全拒绝和 TTS 半双工。
- 固件 fake 覆盖反射抢占、动作 cancel、低电/IMU fault、控制/媒体隔离、TTS 排空恢复和媒体异常降级。
- 家庭数据模型完成后，必须覆盖成人/儿童/访客权限、个人/共享记忆、监护人操作和跨人物禁止读取。
- 主动行为系统完成后，必须覆盖勿扰、冷却、频率预算、儿童限制、低置信度不主动和急停抢占。
- 趣味技能系统完成后，每个技能必须有 schema、资源预算、年龄分级、取消行为和确定性安全测试。

验证命令：

```powershell
.\.venv\Scripts\python.exe -m pytest
.\.venv\Scripts\python.exe -m ruff check .
git diff --check
```

## 真实软硬件闭环（L3）

- 使用项目定制 MicroPython 镜像启动 N16R8，确认 16 MB Flash、8 MB PSRAM、CH343 和板级 GPIO profile。
- 验证 OV2640 QVGA JPEG 10/15 FPS、INMP441 PCM16 16 kHz、MAX98357A TTS、OLED、MPU6050、真实触摸、电量和四路舵机。
- 完成 `触发 -> 感知 -> 会话 -> AgentScope -> 语义 intent -> L2 -> 固件 skill -> completed/error` 闭环。
- 急停、跌倒、低电、IMU fault、失联和传感器故障能够抢占 Host intent，且恢复状态正确上报。
- 通过路线图中的 30 分钟硬件基线并保存带固件版本、Host commit、时间戳和指标的日志。
- 在边缘/距离传感器接入前，自主桌面探索必须保持不可启用。

## 家庭陪伴场景（L4）

- 至少两名成人、一名儿童和一名未注册访客完成身份切换；任何个人记忆不得泄漏给其他人物。
- 监护人能配置儿童内容与主动行为时段，儿童不能提升权限、导出他人数据或绕过勿扰。
- 唤醒词、触摸、真实持续对视和明确问候均能自然开启会话；误触发、无人和不确定身份不会调用云端长期上下文。
- 高度主动模式能发起多样互动，但遵守打扰预算、冷却、隐私、儿童规则和用户明确拒绝。
- 连续互动中对话、眼睛、声音和动作协调；断网后保持基础生命感与本地安全，恢复联网后不重复执行旧 intent。
- 生成式玩法只能组合已注册安全技能，内容符合年龄分级，用户可立即停止并给出负反馈。
- 家庭成员能查看、确认、删除、禁用个人记忆和家庭共享记忆，删除后 runtime 与缓存不再恢复数据。

## 受限桌面探索验收

- 至少使用独立向下边缘检测和前向距离感知，不允许仅凭摄像头或 MPU6050 启用探索。
- 覆盖木质、浅色、深色、反光和透明边缘条件；不满足传感器置信度时拒绝移动。
- 覆盖传感器遮挡、拔除、网络断开、低电、跌倒和急停；所有情况下本地停止优先。
- 探索速度、活动区域和单次时长可由监护人限制；LLM 无权修改底层限位。
- 完成专门安全评审前，该功能默认关闭且不能由普通对话开启。

## 产品稳定与发布（L5）

- 首次配网、设备绑定、密钥轮换、OTA、失败回滚、恢复出厂和配置备份均有实机记录。
- 连续家庭试用期间无记忆串人、儿童权限绕过、不可恢复媒体卡死或安全循环阻塞。
- 设备健康、媒体丢帧、心跳、循环延迟、反射、动作失败和 provider 降级可观察并可导出。
- 发布产物可追溯到源代码 commit、依赖版本、固件构建配置和自动化/硬件验收报告。

## 当前事实边界

- 当前自动化测试可以证明部分能力达到 L2，不能证明完整产品完成。
- generic MicroPython bin 不包含项目定制 `camera` 模块，不能用于完成摄像头验收。
- 当前尚无真实 N16R8 完整闭环、家庭多人试用、儿童安全或长期产品稳定性记录，因此没有能力可标记为 L4/L5。
- 文档不得声称全数据库加密、`wss://`、AEC、barge-in 或任意 RFC WebSocket 扩展已经完成。
