# 路线图

## 路线图原则

naobot 的目标是完整、有趣、适合家庭多人长期使用的陪伴机器人。路线图不再以“代码存在”或“自动化测试通过”宣告产品完成，而按以下成熟度推进：

`L0 缺失 -> L1 原型 -> L2 自动化验证 -> L3 集成可用 -> L4 家庭场景验收 -> L5 产品稳定`

当前逐项成熟度和证据见 `docs/reviews/2026-07-14-product-capability-audit.md`。

## 能力成熟度矩阵

| 能力域 | 当前 | 近期目标 | 产品目标 |

| 陪伴与关系 | L2 | 完成人物/家庭记忆隔离和关系状态 | L5 |
| 自然交流 | L2 | 完成真实 ASR/视觉/TTS 与硬件闭环 | L5 |
| 生命感与主动性 | L1 | 建立主动行为、勿扰、冷却和反馈 | L5 |

| 可靠性与产品化 | L1 | 完成配网、OTA、恢复、看门狗和硬件稳定性 | L5 |
| AI 与扩展生态 | L2 | 建立技能注册、版本协议和评测集 | L5 |

## Phase 1：真实感知与可信身体

- 实现真实触摸、电量、充电状态和低电反射，修复注册触摸确认链路。
- 在 N16R8 上构建并烧录定制 MicroPython，验证 OV2640、INMP441、MAX98357A、OLED、MPU6050、舵机、PSRAM 和 CH343。
- 将显示、蜂鸣器和运动序列全部改为可 tick/cancel，完善 intent 完成回执、去重和有界队列。
- 完成舵机方向、零位、限位、速度和步态校准。

## Phase 2：家庭关系与儿童安全

- 将长期记忆拆为个人、家庭共享和访客域，支持同意、查看、删除、禁用和可解释遗忘。
- 增加监护人、成人、儿童和访客角色，默认儿童最小化持久化并启用内容分级。
- 增加隐私退出、陌生人模式、家庭成员删除和跨设备数据恢复策略。
- 将生产会话写入 `conversation_sessions`，保证人物切换、会话结束和 runtime 生命周期一致。

## Phase 3：自然交流与主动陪伴

- 完成标准 PCM16 16 kHz ASR/TTS 契约、真实目光检测和连续会话闭环。
- 建立情绪、精力、兴趣、动机、打扰预算、勿扰、行为冷却和反馈学习。
- 高度主动行为必须同时满足人物权限、时段、场景置信度、儿童规则和固件安全状态。
- 联网不可用时保留基础生命感、固定趣味互动和本地反射，不承诺开放式离线对话。

## Phase 4：生成式趣味技能

- 建立有版本、参数 schema、资源预算、儿童分级和测试用例的安全技能注册系统。
- 允许 LLM 创作对话、表情、故事和技能编排；L2 负责限幅、重复度控制、内容校验和 fallback。
- 提供可持续扩展的模仿、庆祝、小游戏、家庭仪式和个性偏好学习，不追求固定“99 个游戏”数量。
- Routine 从 CRUD 升级为可解释、待确认、可撤销的触发与执行引擎。

## Phase 5：受限桌面探索

- 增加独立向下边缘传感器和前向距离传感器，完成 GPIO、电源和机械布局验证。
- 建立探索区域、速度限制、定位/里程估计、避障、返回安全区和失效停机。
- LLM 只能提出“探索/靠近/转向”等语义目标，固件探索控制器决定实时路径并拥有拒绝权。
- 在边缘、透明/深色桌面、传感器遮挡、网络断开和低电条件下通过实机安全验收后才能默认启用。

## Phase 6：产品化与长期稳定性

- 完成首次配网、设备绑定、密钥轮换、OTA、回滚、配置备份、自动校准和故障恢复。
- 增加 CI、覆盖率、类型检查、依赖锁定、worker supervisor、看门狗和硬件验收日志。
- 完成家庭连续试用、儿童安全评审、隐私评审和长期运行测试。

## 30 分钟硬件基线门槛

首次 N16R8 集成验收必须在同一台真实设备连续运行 30 分钟并保存带时间戳日志：

- 0 次崩溃、watchdog reset、非计划重启、PSRAM 分配失败和 CH343 串口中断。
- 控制 heartbeat 间隔保持 2 秒；超过 7 秒必须进入离线降级，控制与媒体异常互不拖垮。
- `local_loop_interval_ms` P99 不超过 75 ms，最大值不超过 100 ms。
- 常态视频平均 9-11 FPS，事件窗口平均 13-17 FPS。
- 至少完成 30 轮语音输入/TTS 输出和 10 次断网、重连或服务重启注入。

该门槛只代表 L3 集成基线，不代表家庭场景 L4 或产品稳定 L5。
